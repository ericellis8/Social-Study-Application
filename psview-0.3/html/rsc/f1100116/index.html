For the matrix multiplication serially, I read in the matrices from the file and did all of the calculations using for loops by going through every row and column without threads. Then I used threads and divided up the work depending on how many rows were in the first matrix. If the number of rows in the first matrix divided evenly by the number of threads I used, then I divided the work up evenly among the threads. If there was a remainder of rows at the end of the multiplication, I added those remaining rows to the last thread to do calculations. For the matrices I used doubly nested arrays to represent rows and columns. I also used a semaphore lock in one small part when I was figuring out which starting row each thread would use. I didn't want multiple threads accessing that critical region so when one thread figured out its starting row, then I would add numRowsPerThread and give the next thread a new starting position so every thread was spread out evenly.
