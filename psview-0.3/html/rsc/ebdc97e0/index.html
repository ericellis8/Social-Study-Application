Finite State Machine Representation

The following symbols in the below NFA graphic:

  1.  .* — Represents any character (could be allowed in C or not, depending if the path the .* is on represents an identifier or string) followed by any number of characters.
  2.  ‘    ’ — Represents a space or any number of spaces (i.e. space or tab)
  3.  (0-9) — Represents any number 0-9.  If followed by *, it means any number of digits.
  4.  All circles represent states in the finite state machine.
  5.  All circles with a small circle within them represent final states.

NOTE: Due to the size of the state machine, some of the paths were cut off from scanning the image.

Producing Output by Saving Characters

The nondeterministic finite state machine above can save characters by appending each character to the previous character that is read in.  For example, if the machine were reading in a string such as “int”, the machine would first of all ignore the quotes, then:

  1.  Read in “i”
  2.  Read in “n” and append the “n” to the “i”
  3.  Read in “t” and append the “t” to “in”
  4.  Once the space is reached, the output of the final state will be “int”
  5.  With the output, we can determine if it is in the symbol table or not
  6.  NOTE: After reading in each character, we must ensure that if the next character is not what we want, then we know its going to be an identifier.
